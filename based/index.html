We believe we can help Based move more new installs to a verified first money action.

What we see

Spend, send, and invest products have to make identity checks, funding, and first action feel like one coherent path. Clear sequencing can show which money movements are available at each step, while keeping verification and eligibility easy to understand.

Opportunities to explore

Three places we'd start in the product.

01

Explain verification before wallet funding

We'd start by making the identity check, eligible actions, and funding steps explicit before users try to spend, send, or invest.

Install → Verified account → Wallet ready
02

Route onboarding by intended first action

Worth testing separate first-run paths for spend, send, and invest so users see the requirements for the action they came to complete.

03

Make money-action status easy to recover from

We'd look at clear pending, failed, and completed states for funding and transfers so users know what happened and what to do next.

Ways to measure success

A few ideas for leading indicators we could track with you.

  • ·New installs reaching verified account and wallet-ready status.
  • ·First eligible money action by stated intent: spend, send, or invest.
  • ·Baseline target range set after separating verification, funding, and action completion.
