# Growth brief ‚Äî quality review

Run this checklist **after** the page matches `FORMAT.md`. Use it before publishing or sending the link. Flag failures by section; fix copy or artifacts, not the arc.

## Outcome headline (quantified)

- [ ] States a **measurable outcome** you could track, not a vibe or strategy summary.
- [ ] Uses a **specific quant** (rate, count, ratio, per-100). **No `about`, no `~`, no deadline in the headline** (`within 90 days`, `by Q3`).
- [ ] Quant shape fits the company (not the same ‚Äúlift by X%‚Äù line on every brief unless that is the honest read).
- [ ] Framed as **We believe we can help‚Ä¶**, not ‚Äúyou need to.‚Äù
- [ ] You could name the metric in a dashboard without rewriting the sentence.

**Fail:** `‚Ä¶lift conversion by about 10%.` / `‚Ä¶unlock growth.` / `‚Ä¶at an inflection point.`

## Insight (`What we see`)

- [ ] Eyebrow is **`What we see`**, not `Our read` or other internal labels.
- [ ] Reads like a **talking point to them** (`you` / `your`), not an internal brief for Upstory.
- [ ] Does **not** recap their news, funding, or launch they already know.
- [ ] Does **not** open with ‚Äú{event} is an opportunity to create value.‚Äù
- [ ] One product-sharp observation + wedge + bridge to opportunities (max 3 sentences).
- [ ] **Unique to this client** (swap company name and it should read false).

## Voice and tone

- [ ] **Not patronizing** (no ‚Äúyou may not realize,‚Äù no explaining their business back to them).
- [ ] **Not condensing** (no slide-deck fragments cramming three ideas into one line).
- [ ] **Peer, not auditor** (senior designer who looked at the product, not a rubric).

## AI phrasing (hard fail)

Scan the full page for these patterns. Rewrite any hit.

### Punctuation and structure

- [ ] **No em dashes** in page copy. Use periods, commas, or a second sentence.
- [ ] **No contrast pivots:** ‚Äúit‚Äôs not X, it‚Äôs Y‚Äù / ‚Äúnot X but Y‚Äù / ‚ÄúX, not Y‚Äù at the end of a clause.  
  **Fail:** `Cash Back activation within a week of first payment, not a month out.`  
  **Fix:** `Cash Back activation within seven days of first payment.`
- [ ] **No trailing ‚Äúnot ‚Ä¶‚Äù** corrections (`not generic`, `not a month out`, `not buried in settings`). State what you want positively.

### Banned words (page copy)

- [ ] No **actually**, **real** (except the hyphenated category **real-world** in internal docs only; avoid on the page), **something real**, **a real [noun]**, **genuinely**, **truly** as emphasis.
- [ ] No banned buzzwords from `SKILL.md` (leverage as noun, synergy, unlock, ecosystem, best-in-class, deep dive, move the needle, etc.).
- [ ] No generic AI transitions (‚ÄúIn today‚Äôs landscape,‚Äù ‚ÄúIt‚Äôs worth noting,‚Äù ‚ÄúThis is where X comes in‚Äù).
- [ ] No closer fluff (‚Äúhappy to chat,‚Äù ‚Äúif helpful,‚Äù ‚Äúwould love to‚Äù).
- [ ] The word **trust** does not appear in external-facing copy.

## Specificity

- [ ] Names a **product surface, channel, cohort, or moment** (not ‚Äúthe user journey‚Äù alone).
- [ ] Headline number ties to something defensible from the CSV row (or labeled conservative in measure bullet 3).
- [ ] Opportunities map to **Outreach Insight** and **Hiring Focus**, not generic trust-type boilerplate pasted verbatim.

## Artifacts

Run the **relevance test** in `FORMAT.md` for every visual. **Omit** if any check fails (nothing beats a wrong chart).

- [ ] Artifact type matches the opportunity: **journey** = sequence/handoff; **funnel** = stage drop-off; **calculator** = sizing upside with sliders; **chips** = rare tag set; **none** = cohort/analytics/copy-only.
- [ ] **Calculator** only when inputs are defensible and the block is about quantified upside; helper text says assumptions; not presented as their real data.
- [ ] **0‚Äì2 artifacts** on the page is normal; three only if each block passes the test independently.
- [ ] Highlighted step or key funnel stage = the moment the title says you would fix.
- [ ] Labels are **client-specific**; swapping this visual to another block would break meaning.
- [ ] Removing the visual loses order or relative drop the prose does not state.
- [ ] No filler because ‚Äúevery brief needs a funnel on 02.‚Äù
- [ ] Illustrative funnels labeled; no fake precision as their data.

## Structure and layout

- [ ] Exactly six blocks in order: header ‚Üí headline ‚Üí insight ‚Üí opportunities ‚Üí ways to measure ‚Üí pitch.
- [ ] No extra sections (fit essay, second hero, solution essays).
- [ ] `What we see` layout: lightbulb + eyebrow on one row; body left-aligned below.
- [ ] Opportunity spacing reads comfortable (not cramped between `01` / `02` / `03`).

## Measurement block

- [ ] Three bullets only; same **primary metric** as the headline.
- [ ] Bullet 1 = north-star + window; bullet 2 = cohort/isolation; bullet 3 = readout/threshold.
- [ ] Lead line matches default unless there is a deliberate, reviewed reason to change it.

## Pitch

- [ ] Upstory closer copy is **verbatim** locked text.
- [ ] Client value ends above the closer; pitch does not repeat opportunity content.

## Measurement bullets

- [ ] Each bullet is **positive** (what you would track), not a contrast pair (`X, not Y`).
- [ ] Same primary metric family as the headline.

## Quick read-aloud test

Read the page aloud in under 90 seconds. If anything sounds like ChatGPT (contrast tails, ‚Äúactually,‚Äù ‚Äúreal,‚Äù em dashes), fix it before ship.
